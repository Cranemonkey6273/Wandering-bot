Wandering Bot mobile app direction

Style board and app layout choices

The app should feel like a serious DayZ server control surface, not a marketing page. These are four usable visual directions for the dashboard, mobile app, and public site. The aim is cleaner, more adult, less noisy, and still recognisably Wandering Bot.

Recommendation: start with Command Center if you want to keep the current brand. Pick Field Tablet if you want it calmer and more premium. Signal Ops is best for a live-alert heavy product. Steel Console is strongest for billing, owners and business users.

Suggested App Pages

1. App HomeCurrent server, server switcher, players online, Nitrado/ADM/FTP status, next restart, next raid window, urgent warnings.
2. Recent FeedsKillfeed, builds, flags, placed items, connections, damage, radar pings and pinned board previews filtered by Cherno, Livo or all servers.
3. Shop and EconomyQuick edits for shop items, categories, prices, stock, player balances, wages, rewards and global economy settings.
4. EventsAirdrops, animal drops, zombie hordes, live spawn tracker and restart-needed status. Keep risky XML actions behind confirmed backend forms.
5. Maps and ZonesHeatmaps, radar zones, safe zones and PVP zones with a clear server picker before editing anything.
6. Preset FilesDownload ready-made vanilla, unlimited stamina, build-anywhere and boosted economy files in the correct format for phone upload or guarded backend apply.
7. ControlsRestart schedules, vehicle reset schedules, raid weekend controls, base damage, container damage and server action audit state.
8. MembersBan, unban, warn, linked-gamertag lookup, access review, staff notes and role checks scoped to the selected server.
9. Billing and AccessPlan, renewal, upgrade, customer portal, admin logins, temporary staff access and private/manual access grants.
10. Guide LibraryBeginner guide, download text, file definitions, helpful tools, events, animal/zombie spawns, loot in buildings, custom categories, tags and tiers.

Design Rules I Would Use

Keep

  • One obvious server picker on every tool that can affect Cherno, Livo or another customer server.
  • Small, dense dashboard cards for repeated use on phones.
  • Real feed previews and real feature names from the bot.
  • Separate owner console from customer server tools so billing does not feel mixed into gameplay settings.
  • Clear warning states before preset applies, restarts, vehicle resets and billing access changes.

Avoid

  • Huge hero sections inside the actual dashboard app.
  • Too many emoji separators or decorative labels in controls.
  • Fake trust numbers once billing is live. Use real counters or say "sample" clearly.
  • Putting dangerous actions behind tiny icon-only controls.
  • Full arbitrary XML editing on mobile. Use tested preset downloads and guarded backend apply instead.
  • Any phone-side secret handling. Stripe secret keys, Nitrado tokens and Discord tokens stay backend-only.
